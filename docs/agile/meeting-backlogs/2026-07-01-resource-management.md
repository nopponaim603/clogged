# Meeting Note รอบที่ 3 - Resource Management

**วันที่:** 2026-07-01

## 1. แนวเกมโดยรวม
- เกมแนว Resource Management แบบ Turn-based
- มีแผนที่ (map) ที่ทรัพยากรและตำแหน่งถูกสุ่มใหม่ทุก Turn
- ผู้เล่นควบคุม Core Character เพื่อรวบรวมทรัพยากรจากแผนที่
- แต่ละ Turn แบ่งออกเป็น 2 ช่วงหลัก:
  - ช่วง a: การหาของและรวบรวมทรัพยากร (ใช้ Action Point(s))
  - ช่วง b: การต่อสู้
    - เป้าหมายของเกมยังไม่แน่ชัด — อยู่รอด 30 วัน?

## 2. ข้อเสนอรูปแบบเกม (Brainstorm 3 แนวทาง)

### 2.1 รูปแบบ A (ไอซ์)
- Core Character: ยานที่มีลูกเรือ (Crew) และ Facility ต่างๆ ให้ลูกเรือ, ตัวละครแนวหุ่นยนต์
- Theme: Post-apocalypse
- Stats พื้นฐาน: ATK, VIT (HP/DEF), SPD (ความเร็ว + โอกาส Bonus Action), Con (Cri-rate), SUP (เพิ่ม bonus ให้ stat อื่น)
- ใช้ลูกเต๋าในการคำนวณ Action Point และทอยลูกเต๋าเพื่อลด HP ศัตรูตามหน้าเต๋าที่ทอยได้
- มีระบบ Training (ใช้ทรัพยากรพัฒนาตัวละคร) และระบบ Gear (ใส่เพื่อเพิ่ม stat)

### 2.2 รูปแบบ B (ปาร์ค)
- Core Character: ถ้าเป็นยาน อยากให้มีเกราะและป้อมปืน, หรือแบบมนุษย์
- ระบบต่อสู้:
  - **แบบยาน:** แนว Bullet Hell — เข้าด่านโจมตีลูกสมุน (ฆ่าแล้วได้ Buff) ก่อนเข้าบอส, เน้น Upgrade ยาน (ATK/HP/DEF)
  - **แบบมนุษย์:**
    - แนว Vampire Survivors (อัพเกรดก่อนเข้าด่าน ความสามารถใช้ได้เฉพาะด้านนั้น)
    - แนว Tower Defense (Upgrade ตัวละคร + ฐาน, มีป้อมยิง, ผู้เล่นสนับสนุนด้วยป้อม ส่วนตัวละครสู้เองอัตโนมัติ)

### 2.3 รูปแบบ C (เอก) — V.2 ปรับเข้าเป็นเส้นตรงมากขึ้นและใช้ stat แทน
- Core Character: ได้หมด (ไม่จำกัดรูปแบบ)
- ผู้เล่นควบคุม Unit ทั้งหมด 4 ตัว โดยใน 1 Turn จะเลือกหน้าที่ให้แต่ละตัว (พัฒนา stat หรือ ทำเควส)
- Unit ทุกตัวมี 4 stat โดยแต่ละตัวจะมี 1 stat ที่โดดเด่นและไม่ซ้ำกับตัวอื่น
- 1 Turn แบ่งออกเป็น 2 ช่วง (มี 4 ช่อง/ตำแหน่งให้วาง unit ในแต่ละช่วง):
  - ช่วง a: พัฒนา unit — เพิ่ม stat ต่างๆ ให้ unit ตามช่องที่ว่าง เพื่อให้มี stat เพียงพอผ่านเควสในช่วง b
  - ช่วง b: รับเควส — เพิ่ม Progress ของเกม โดยมีเควสให้เลือก 1 ใน 4 (3 เควสสุ่มทุกรอบ + 1 เควสหลักที่อยู่เสมอ)
- Unit 1 ตัวทำได้แค่ 1 ช่วง ต่อ 1 Turn (ทำสองช่วงพร้อมกันไม่ได้)
- ผ่านเควสโดยเปรียบเทียบค่าพลังรวมของ unit ที่นำไปทำเควส กับค่าพลังที่เควสระบุ (ถ้าค่าพลังรวม ≥ ค่าที่กำหนด ผู้เล่นผ่านเควสนั้น) — ต้องผ่านเควสหลักทั้ง 3 เควสเพื่อจบเกม

## 3. สรุป

### 3.1 Core Loop
ทุก Turn ประกอบด้วย 2 ช่วง:
1. **ช่วงพัฒนา** — ผู้เล่นจัดสรรทรัพยากรที่มีจำกัดเพื่อพัฒนาหรือรวบรวม → ผลลัพธ์: เพิ่ม stat, ได้ทรัพยากร, upgrade Core Character หรือ unit
2. **ช่วงต่อสู้** — ผู้เล่นนำสิ่งที่พัฒนาได้ไปทำเควสหรือต่อสู้ → ผลลัพธ์: ได้ Progress เกม, รางวัล, หรือล้มเหลว (เสียทรัพยากรบางอย่าง หรือ unit ใช้งานไม่ได้)

### 3.2 Objective — เป้าหมายของเกม (3 แนวทางที่พิจารณา)
1. **แบบที่ 1:** มี 1 เควสหลักอยู่ตลอด + 3 เควสย่อยแบบสุ่มทุก Turn (เควสย่อยให้ทรัพยากร/อุปกรณ์/Buff) — ผู้เล่นชนะเมื่อผ่านเควสหลักครบ 3 ครั้ง (ไม่จำเป็นต้องต่อเนื่อง)
2. **แบบที่ 2:** รวบรวมทรัพยากร/upgrade จนถึงเกณฑ์ที่กำหนด (เช่น stat รวมถึงระดับ X หรือมี Gear ครบชุด) — ผู้เล่นชนะเมื่อถึงเกณฑ์ + ผ่านเควสหลักอย่างน้อย 1-2 ครั้ง
3. **แบบที่ 3:** ผู้เล่นต้องสู้กับบอสที่โผล่มาทุกๆ Turn ที่กำหนด — กำจัดบอสเพื่อเพิ่ม Progress ให้ครบตามที่กำหนดเพื่อชนะเกม

### 3.3 Resources
- ทรัพยากรสุ่มบนแผนที่
- stat เป็นทรัพยากรที่พัฒนาได้
- Progress Token ที่บอกความก้าวหน้าของเกม (3 Token = ชนะ)

### 3.4 Lose Conditions
- เมื่อ Core Character ถูกทำลาย
- เมื่อไม่สามารถผ่านเงื่อนไขการชนะได้ภายใน Turn ที่กำหนด
