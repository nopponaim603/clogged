## เฟส 0 — Pre-production (W1–W2 · 29 มิ.ย.–12 ก.ค.)

**ออกจากร่างไอเดียให้เป็นแผนที่ทำได้จริง — ห้ามข้าม**

- [ ] ตอบคำถามค้างใน [Idea-design.md](Idea-design.md) ข้อ 5 ให้ครบ:
  - Core loop, Win/Lose condition, Progression, โครงสร้างด่าน (level vs sandbox)
  - Setting/Theme, Platform & Control, Target audience, Art/UI direction
- [ ] **ล็อก MVP scope** — ระบุระบบขั้นต่ำที่ยังสนุก (กันงานบานปลาย ⚠️ สำคัญสุด)
- [ ] เขียน core loop เป็นไดอะแกรม 1 หน้า
- [ ] **Tech setup:** ล็อก Unity version, render pipeline (URP/Built-in), โครงสร้างโฟลเดอร์, coding convention
- [ ] ตั้ง git workflow (branch, PR, commit convention) + แบ่งบทบาทในทีม

**Milestone:** GDD ฉบับล็อก + โปรเจกต์ Unity ตั้งเสร็จ commit ขึ้น repo ได้

## เฟส 1 — Prototype / Vertical Slice (W3–W6 · 13 ก.ค.–9 ส.ค.)

**เป้าหมายเดียว: ตอบให้ได้ว่า "เกมนี้สนุกไหม" ด้วย grayblock (ยังไม่ต้องสวย)**

- [ ] ระบบ **การไหลของทรัพยากร** พื้นฐาน (resource node → ท่อ/สายพาน → ปลายทาง)
- [ ] **โหนดการผลิต 1 แบบ** — รับวัตถุดิบ ผลิตของใหม่ + เกิดเศษเหลือ (by-product)
- [ ] ระบบ **ตรวจจับการตัน/ขาด** (clog / starvation) + การเตือนพื้นฐาน
- [ ] ด่านทดลองเล่นได้ 1 ด่าน (วาง/รื้อ/เชื่อมต่อระบบได้)
- [ ] ทดลองเล่นกันเองในทีม

**🚦 Decision gate (สิ้น W6):** core loop สนุกพอไหม?
→ ถ้าสนุก: ไปเฟส 2 · ถ้ายัง: ปรับดีไซน์ก่อน (มี buffer ในแผน)

## เฟส 2 — Production: ระบบหลัก (W7–W12 · 10 ส.ค.–20 ก.ย.)

**สร้างระบบให้ครบตาม MVP — ยังเน้น function มากกว่าความสวย**

- [ ] **Production pipeline เต็ม** — ต่อสายการผลิตหลายขั้น, รวมวัตถุดิบแบบ "สมการ", จัดการ waste
- [ ] **ระบบ currency + อัพเกรด** ทรัพยากร/เครื่องจักร
- [ ] **ระบบ Idle / Clicker** — หาทรัพยากรพื้นฐาน
- [ ] **โครงสร้างด่าน / progression** — ปลดล็อกตามลำดับ, โจทย์ยากขึ้น
- [ ] **Save / Load**
- [ ] **UI/UX** เน้น readability ของระบบ (สถานะท่อ, การเตือนขาด/ตัน ครบทั้ง 2 กรณี)

**Milestone (สิ้น W12):** Alpha — ฟีเจอร์ครบ เล่นจบลูปได้ตั้งแต่ต้นจนจบ

## เฟส 3 — Content & Polish (W13–W16 · 21 ก.ย.–18 ต.ค.)

- [ ] สร้าง **เนื้อหา/ด่าน** ตามจำนวนที่วางไว้
- [ ] **Art pass** — แทน grayblock ด้วย asset จริง, สไตล์ภาพ
- [ ] **เสียง + juice/feedback** (เสียงตัน, เอฟเฟกต์, animation)
- [ ] **Tutorial / onboarding** สอนผู้เล่นใหม่
- [ ] **Balancing** — ปรับความยาก, อัตราการผลิต, ราคาอัพเกรด

**Milestone (สิ้น W16):** Beta — content ครบ เล่นได้ครบทั้งเกม

## เฟส 4 — Test & Release (W17–W18 · 19 ต.ค.–1 พ.ย.)

- [ ] **Playtest กับคนนอกทีม** + เก็บ feedback
- [ ] **แก้บั๊ก** (จัด priority) + optimize performance
- [ ] ทำ **build จริง** ตาม platform ที่เลือก
- [ ] เอกสารส่งมอบ / หน้าเผยแพร่ (ถ้ามี)

**Milestone (สิ้น W18):** Gold — release build ส่งมอบได้

## หลักการบริหารแผน

- **เผื่อ buffer:** อย่าอัดงาน 100% ทุกสัปดาห์ — เผื่อเวลาแก้ของที่พลาด
- **MVP มาก่อนเสมอ:** ฟีเจอร์เสริมทำเมื่อ MVP เสถียรแล้ว
- **Playtest บ่อย:** ทดลองเล่นทุกสิ้นเฟส ไม่รอจบเกม
- **Standup สั้นๆ:** เช็คความคืบหน้าทุกสัปดาห์ ปรับ scope ทันทีถ้าช้า
- **ระวัง scope creep:** ⚠️ ความเสี่ยงอันดับหนึ่งของเกม systems-driven — ทุกฟีเจอร์ใหม่ต้องถามว่า "อยู่ใน MVP ไหม"
