# [Archived] Idea & Design Draft — *clogged*

> **สถานะ:** เก็บถาวรเพื่ออ้างอิงประวัติ — เนื้อหานี้คือร่างไอเดียตั้งต้นก่อนทีมจะปรับทิศทางจริงตาม prototype (ดู [Design Pivot](../../gdd/00-concept.md#2-️-design-pivot--บันทึกไว้เพื่อความชัดเจน))

> **Code name:** clogged

---

## 1. ภาพรวม (Vision)

อยากได้เกมที่ **เน้นการออกแบบและพัฒนาระบบ** (systems-driven) มากกว่าการเน้นเนื้อเรื่องหรือกราฟิก
แก่นหลักคือ **การบริหารทรัพยากรที่มีอยู่อย่างจำกัด** ให้ผ่านด่าน/เติบโต โดยผู้เล่นต้องวางแผน จัดสรร และต่อยอดระบบที่ซับซ้อนขึ้นเรื่อยๆ

> **ตีความชื่อ "clogged" (อุดตัน):** ใช้เป็น theme หลักได้เลย — เกมว่าด้วย "การไหลของทรัพยากร" ที่ต้องไม่ให้ **ตัน**
> เช่น ระบบท่อ/สายพาน/โรงงานที่ถ้าวางแผนผิด ทรัพยากรจะติดขัด ล้น หรือขาด → เป็นทั้งชื่อ ธีม และ core challenge ในตัวเดียวกัน

## 2. แนวทางที่กำลังพิจารณา (Design Pillars / Genres)

### 2.1 Resource Management *(แกนที่มาแรงที่สุด)*
- บริหาร assets ที่มีอยู่อย่างจำกัดให้ผ่านด่านได้
- เป็น **puzzle resource management** — ใช้ assets ที่มีอยู่แก้โจทย์เพื่อผ่านด่าน
- มี **currency พิเศษ** บางอย่างที่หามาเพื่อใช้ผ่านด่าน / อัพเกรดทรัพยากร
- **Fun factor:** ความรู้สึก "จัดสรรของน้อยให้พอ" และวางแผนล่วงหน้า

### 2.2 Factory Management
- คล้ายการประกอบ **สมการเคมี** — รวมวัตถุดิบเข้าด้วยกันเพื่อสร้างของใหม่
- การผลิตมี **เศษเหลือ (by-product / waste)** เกิดขึ้นด้วย → ต้องจัดการของเหลือ
- **Fun factor:** ความสุขจากการต่อ pipeline การผลิตและ optimize ให้มีประสิทธิภาพ

> เศษเหลือนี้แหละที่เชื่อมกับธีม "clogged" ได้ — ถ้าไม่จัดการ ของเสียจะอุดตันระบบ

### 2.3 Survival Management
- ยังเป็นไอเดียเปล่า — *ต้องขยายความ*
> คำถามที่ต้องตอบ: อยู่รอดจาก **อะไร**? (เวลา / ทรัพยากรหมด / ภัยคุกคามภายนอก?) และเงื่อนไข "ตาย" คืออะไร

### 2.4 Idle Game
- **Fun factor:** การลงทุน และความรู้สึก "เติบโตขึ้นเรื่อยๆ ไม่ติดขัด"
- **ข้อกังวล:** goal / end game **ควบคุมยาก** — เกม idle มักไม่มีจุดจบที่ชัด
> แนะนำใช้เป็น **ระบบเสริม** (เช่น ผลิตทรัพยากรพื้นฐานตอนไม่ได้เล่น) มากกว่าเป็นแกนหลัก

### 2.5 Clicker
- คลิกเรื่อยๆ เพื่อได้ resource มาอัพเกรด
- **Fun factor:** feedback loop ที่ตอบสนองทันที
> เหมาะเป็นกลไก "หาทรัพยากรเริ่มต้น" หรือ active boost มากกว่าจะเป็นแกนทั้งเกม

## 3. ทิศทางที่แนะนำ (Recommended Direction) — ณ ตอนเขียนร่างนี้

จากไอเดียทั้งหมด แนวที่ "เน้นออกแบบระบบ" และเข้ากับชื่อ **clogged** ที่สุดคือ:

> **Resource Management + Factory** เป็นแกนหลัก
> โดยมี **Idle / Clicker** เป็นระบบเสริมในการหาทรัพยากร และยืมความรู้สึก "เติบโต"

**ลูปหลักที่เสนอ (ร่างเดิม):** หาวัตถุดิบ → ป้อนเข้าสายการผลิต → จัดการเศษเหลือไม่ให้อุดตัน → ได้ผลผลิต/currency → อัพเกรดระบบ → รับโจทย์ที่ยากขึ้น

> ⚠️ **ทิศทางนี้ไม่ใช่สิ่งที่ prototype สร้างจริง** — ทีมปรับไปเป็น Resource Management + Action Survival แทน ดู [GDD Concept](../../gdd/00-concept.md)

## 4. ฟีเจอร์/ระบบที่ระบุไว้แล้ว (ในร่างเดิม)

- **ระบบเตือนเมื่อทรัพยากรขาด/ไม่ครบ** — แจ้งผู้เล่นเมื่อ resource หายไปหรือไม่พอ
> ขยาย: ควรเตือนทั้งกรณี **ขาด (starvation)** และ **ล้น/ตัน (clogging)** เพื่อสื่อธีม
> รูปแบบการเตือน: ไอคอนบนเครื่องจักร / สีของท่อ / เสียง / สรุปสถานะรวม

## 5. ส่วนที่ยังขาด — ต้องตัดสินใจ (ในร่างเดิม)

| หัวข้อ | คำถามที่ต้องตอบ |
|---|---|
| **Core Loop** | หนึ่งรอบการเล่นทำอะไรซ้ำๆ และอะไรทำให้สนุกในรอบที่ 100? |
| **Win / Lose Condition** | ผ่านด่านเมื่อไหร่? แพ้/จบเกมเมื่อไหร่? |
| **Progression** | ผู้เล่นเก่งขึ้น/ระบบใหญ่ขึ้นอย่างไร |
| **โครงสร้างด่าน** | level-based หรือ sandbox/endless? |
| **Setting / Theme** | เรื่องราว/บริบทคืออะไร — โรงงาน? ท่อประปา? ร่างกาย? โรงบำบัด? |
| **Platform & Control** | PC / mobile? |
| **Target Audience** | casual / hardcore optimizer |
| **Art / UI Direction** | เน้น readability ของระบบเป็นหลัก |
| **MVP / Scope** | เวอร์ชันเล่นได้ขั้นต่ำมีระบบอะไรบ้าง |

> สถานะปัจจุบันของแต่ละข้อ ดูที่ [GDD — Mechanics, ส่วนที่ยังต้องตัดสินใจ](../../gdd/01-mechanics.md#ส่วนที่ยังต้องตัดสินใจ-ต่อยอดจาก-idea-design-เดิม)
